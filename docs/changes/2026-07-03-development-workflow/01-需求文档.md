# 需求文档：固定化开发与调试流程

## 目标

让 `variants/srad/ours_192lane/` 的代码修改、性能优化、编译故障排查有一套可提交到 GitHub 的固定流程，协作者拉仓库后可一比一复用。

## 范围

- 项目本地 skills：`.agents/skills/`。
- 模糊输入自动触发文档化：`plan-docs`，并兼容用户写成 `plan-docx`。
- Spec/Kiro/大文件读取流程：`spec-subagents`。
- 长代码索引：`docs/CODEMAP_ours_192lane.md`。
- 调试手册：`docs/DEBUG_PLAYBOOK.md`。
- 开发流程：`docs/DEVELOPMENT_WORKFLOW.md`。
- Kiro/spec 规格：`.kiro/specs/development-workflow/`。
- 可运行检查：`scripts/check_ours_192lane.py` 与测试。

## 验收

- 新协作者能先读项目本地 skill，不依赖全局技能安装。
- 新协作者能运行 `python scripts/validate_project_skills.py` 验证本地 skills。
- 改代码前有红绿 TDD 入口。
- 编译 bug 先定位层级和根因，不盲改。
- 完成前有统一验证命令。
