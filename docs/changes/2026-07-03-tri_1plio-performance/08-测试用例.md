# 测试用例

## T1：源码配置一致性

- 目标：确认 `tri_1plio` 源码、Makefile、host、run.sh、测试断言一致指向 256*256*64。
- 命令：`python -m unittest variants.stencilflow.tri_1plio.tests.test_board_volume_config`
- 验收：测试通过，且 `iter_cnt=16384` 路径无自相矛盾配置。

## T2：基线同口径计时

- 目标：`tri` 与 `tri_1plio` 打印同一组计时字段。
- 验收：同一输入、同一 xclbin 运行方式下能比较 `pipeline_total_us`。

## T3：小数据 emu 回归

- 目标：确保优化后不破坏 256*20 小数据路径。
- 命令：在对应 variant 下运行 `make sim`。
- 验收：输出通过现有 compare 或人工指定脚本，且计时不显著倒退。

## T4：大数据上板验收

- 目标：256*256*64 上板运行。
- 命令：在板端运行 `./run.sh <xclbin> 16384 ./board_data/input.txt <out.txt>`。
- 验收：`tri_1plio` 快于 `tri` 30.9ms 基线；期望小于 28ms。

## T5：瓶颈对照实验

- 目标：分别验证 fanout、FIFO depth、pack 格式是否为瓶颈。
- 验收：每次只改一个因素，记录板上时间、正确性、是否可复现。
