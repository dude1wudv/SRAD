# 计时口径

目标是排除“仿真快、上板慢”中由计时范围不一致造成的假反转。

当前证据：

- `tri/ps/host.cpp` 在 `topStencil.run()` 后只围绕 `toppl(...).wait()` 打印 `pl_total_us`。
- `tri_1plio/ps/host.cpp` 从 `topStencil.run()` 前开始计 `aie_run_us`，并单独打印 `pl_transfer_us`。
- 两边默认输出字段不一致，不能直接证明 30.9ms 与 33.9ms 是完全同口径。

建议改动：

- 两边统一保留 `pl_transfer_us`、`aie_run_us`、`pipeline_total_us`。
- 计时边界固定为：graph run 发起前，到 TopPL 与 graph 都完成后。
