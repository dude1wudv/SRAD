# 参数规模

当前最大风险是目标规模与当前源码不一致。

目标规模：

- 行宽：256
- 行数：256
- 深度：64
- `iter_cnt = 256 * 64 = 16384`
- DDR words：`16384 * (256 / 16) = 262144`

当前 `tri_1plio` 源码：

- `Makefile`: `BOARD_GRID_ROWS ?= 256`、`BOARD_GRID_DEPTH ?= 64`、`BOARD_DATA_ITER ?= 16384`
- `pl/TopPL.cpp`: `TRI1PLIO_MAX_ITER 16384`、`TRI1PLIO_MAX_DDR_WORDS 262144`
- `ps/host.cpp`: `BOARD_GRID_ROWS = 256`、`BOARD_GRID_DEPTH = 64`
- `run.sh`: 默认 `ITER=16384`

结论：

- 256*256*64 板上默认规模已统一到当前仓库。
- 仍需在 Vitis/板上重新构建验证。
