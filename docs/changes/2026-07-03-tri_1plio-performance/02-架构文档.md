# tri 与 tri_1plio 架构对照

## 相关目录

- 基线：`variants/stencilflow/tri`
- 优化目标：`variants/stencilflow/tri_1plio`
- 对照候选：`variants/stencilflow/tri_8plio`、`variants/stencilflow/tri_16plio`

## tri 数据流

PL:

1. DDR 512-bit word 读取。
2. 拆为 128-bit PLIO word。
3. AIE 输出 128-bit word 重新合并为 DDR 512-bit word。
4. 写回 DDR。

AIE:

1. 单输入 PLIO 连接到 `StencilCoreGraph::in`。
2. `StencilCoreGraph::in` 同时连接 lap 与 flux1。
3. lap 产生 4 路 `kLapForwardElems`。
4. flux1 产生 5 路 `kFluxInterElems`。
5. flux2 合成输出。

## tri_1plio 数据流

PL:

1. DDR 512-bit word 读取。
2. 拆为 128-bit PLIO word。
3. AIE 输出 128-bit word 重新合并为 DDR 512-bit word。
4. 写回 DDR。

AIE:

1. 仍是 1 个输入 PLIO。
2. `TopGraph.h` 将同一 PLIO 输出 fanout 到 `core.in[0]`、`core.in[1]`、`core.in[2]`。
3. lap 输出 `sub_pack`，同时供 flux1 和 flux2 使用。
4. flux1 输出 `mask_pack`。
5. flux2 读取 `mask_pack`、原始输入窗口、`sub_pack` 后合成输出。

## 性能反转候选原因

- 计时口径：`tri` 只打印 `pl_total_us`，`tri_1plio` 打印 `pl_transfer_us` 与 `aie_run_us`；两个数是否同口径需要统一。
- 参数规模：当前源码默认规模与 256*256*64 不一致，可能出现“上板版本”和“当前源码”不一致。
- 单 PLIO 三路 fanout：小数据下内部中间流减少会更快；大数据上板时，单 PLIO 广播和 FIFO backpressure 可能变主瓶颈。
- FIFO 深度：当前 `tri_1plio` FIFO depth 为 6，测试文件仍断言 2；配置来源需要清理。
- AIE 包格式：`tri_1plio` 减少 flux1->flux2 数据量，但新增 lap->flux2 fanout 与 flux2 原始输入窗口消费。

## 已实施结构修正

- `tri_1plio` 已改为 256*256*64 板上默认规模：`iter=16384`、DDR words `262144`。
- host 已移除 `toppl(input_bo, output_bo, iter_cnt, 1)` 中不存在的 `repeat` 参数。
- lap 的 `sub_pack` 已拆成两个独立输出端，分别连接 flux1 与 flux2，避免同一个 AIE 输出端在 graph 层 fanout。
