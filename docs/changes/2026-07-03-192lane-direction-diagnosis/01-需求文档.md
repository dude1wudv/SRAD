# 01-需求文档：192lane 方向诊断

## 背景

`variants/srad/ours_96lane/` 已在板上跑通；`variants/srad/ours_192lane/` 来自把 96lane 扩成 192lane，但遇到明显布局布线压力，硬件编译不成功。

## 本次目标

只做方向复盘：比较 `ours_96lane` 与 `ours_192lane` 的关键配置、AIE/PL 连接、资源映射和轻量检查结果，判断“96 复制成 192”是不是错误方向。

## 范围

- 只读诊断，不修改实现代码。
- 重点文件：`aie/Config.h`、`aie/ProcessGraph/StencilCoreGraph.h`、`aie/ProcessUnit/srad_coeff_update.cc`、`pl/TopPL.cpp`、`pl/Q0Ctrl.cpp`、`conn.cfg`、`Makefile`、`pl/README.md`。
- 没有本地 `_x` / impl / route 日志，因此不声称已经定位到唯一失败 error line。

## 需求结论

当前证据支持：`ours_192lane` 不是“96lane 的安全线性翻倍”，而是把 AIE 核数、PL CU 数、AIE-PL stream 形态、q0 控制扇入扇出一起推高。方向上应先停止继续硬怼 192 布局布线，改为做更小的架构验证或回退到 96 稳定路线。
